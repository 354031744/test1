支付接口

接口描述

  • 接口请求应以HTTPS POST form-data方式提交数据

接口参数定义

  • 交易请求地址: https://www.xlemonpay.com/api/pay/pay.php
  • 请求参数中的参数编码为UTF-8
参数 参数名称 参数类型 使用 说明
公共参数
merchant_code 商家号 string 必填 由柠檬支付分配的商家号，如：20181001
notify_url 服务器异步通知地址 string 必填 支付成功回调地址，必填。订单成功支付后，由平台的回调机制主动通知商户，告知支付订单已支付。
interface_version 接口版本 string 必填 接口版本，固定值：1.0
client_ip 客户端iP string 必填 消费者创建交易时所使用机器的IP或者终端ip，最大长度为15个字符。如：192.168.2.35
sign 数据签名 string 必填 签名数据，具体请见附录的签名规则定义
product_name 商品名称 string 必填 商品名称，不超过100个字符。 如：小米笔记本2
product_desc 商品描述 string 可选 商品描述，不超过300个字符。
product_ext 商品拓展信息 string 可选 产品扩展信息

特定参数

  • 调用上述接口后，用户可根据自己业务支付方式选择以下对应的业务参数，如:微信扫码支付，service_type=weixin_scan。

响应参数

  • 调用上述接口后，青柠支付平台回应商户支付请求。
参数 参数名称 参数类型 使用 说明

message 返回消息 string 选填 返回消息

status 处理结果 string 必填 处理结果,固定值：
1:成功;
0:失败;

支付成功回调函数

接口描述

  • 订单支付结果回调接口，客人支付订单后，柠檬平台将通过主动回调商户回调接口的形式通知商户该笔交易支付结果，默认情况下只有支付成功才会通知。
  • 注意：商户在收到通知并处理完成后必须打印输出“SUCCESS”这七个大写字母，否则，青柠支付系统会认为通知失败，将在随后的一段时间内，以递增的时间间隔重发5次通知。

接口参数定义

参数 参数名称 参数类型 使用 说明
基本参数
merchant_code 商家号 string 必填 由柠檬支付分配的商家号，如：20181001
interface_version 接口版本 string 必填 接口版本，固定值：1.0
sign 数据签名 string 必填 签名数据，具体请见附录的签名规则定义
业务参数
order_id 商户网站唯一订单号 string 必填 商户系统订单号，由商户系统生成,保证其唯一性，最长100位,由字母、数字组成.如：2018012345。
sys_order_id 系统订单号 string 必填 青柠支付系统的订单号。
status 状态 string 必填 1：成功
0：失败
2：其他错误
amount 商户订单总金额 string 必填 该笔订单的总金额，以元为单位，精确到小数点后两位，如：15.03

数据签名

  • 将所有参数按照参数首字母升序排序（空值不参与），最后加上key字段，格式如：param1=val1&param2=val2...&key=商家key
  • 将组装的好的字符串，计算md5，将结果用大写，作为签名数据：sign = md5(data).toUpperCase()
